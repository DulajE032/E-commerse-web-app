
 1. Phase 1: Plan MVP scope
  - Products: earbuds, phones, back covers, chargers, powerbanks
  - Roles: customer, admin
  - MVP features: signup/login, browse products, cart, checkout (cash on delivery first), admin
 product management
 2. Phase 2: Create project structure mkdir uni-store && cd uni-store
  mkdir backend frontend
 3. Phase 3: Setup backend (Node + Express + MongoDB) cd backend
  npm init -y
  npm i express mongoose cors dotenv bcryptjs jsonwebtoken
  npm i -D nodemon
 
 
  - Create folders: config/ models/ controllers/ routes/ middleware/
  - Core files: server.js, config/db.js, .env
  - Add scripts in package.json: "dev": "nodemon server.js"
 4. Phase 4: Backend data models
  - User: name, email, passwordHash, role
  - Product: name, description, price, category, stock, images
  - Cart: user, items[product, qty]
  - Order: user, items, total, status, delivery info
 5. Phase 5: Backend APIs
  - Auth: register/login/profile (JWT)
  - Products: list, details, search, filter by category
  - Cart: add/update/remove/get
  - Orders: create/get user orders
  - Admin: create/update/delete products, update stock/order status
 6. Phase 6: Setup frontend (React) cd ../frontend
  npm create vite@latest . -- --template react
  npm i axios react-router-dom @reduxjs/toolkit react-redux
  npm run dev
 
 
  - Pages: Home, ProductList, ProductDetail, Cart, Checkout, Login/Register, AdminDashboard
  - Global state: auth, cart, products
 7. Phase 7: Connect frontend to backend
  - Axios base URL from .env
  - Store JWT in secure way (httpOnly cookie preferred; localStorage for simple MVP)
  - Protect routes (admin/customer)
 8. Phase 8: Add essential e-commerce quality
  - Validation and proper error responses
  - Stock checks before order creation
  - Image upload (Cloudinary or similar)
  - Basic university-style UI (responsive, clean cards, category chips)
 9. Phase 9: Test and deploy
  - Backend deploy: Render/Railway
  - Frontend deploy: Vercel/Netlify
  - Database: MongoDB Atlas
  - Add CORS + production env vars + build checks
 10. Phase 10: After MVP improvements

 - Online payments (Stripe)
 - Reviews/ratings
 - Order tracking emails
 - Coupon/discount system

Immediate first build target: finish Phases 2–6 with only these features: auth, product
listing, cart, place order. That gives you a usable university store quickly.
